일자를 일부 수정하고 싶으실 때는 깃허브에 접속하셔서 해당 index.html 파일 우측의 연필 모양(Edit) 아이콘을 눌러 텍스트만 고치고 다시 Commit 하시면 실시간으로 반영됩니다. 한번 직접 개설해 보시고 막히는 부분이 있다면 편하게 말씀해 주세요. https://000yoryeong.github.io/profile 나오지 않는데? 보여주신 주소로 접속이 되지 않는 이유는 깃허브 페이지(GitHub Pages)의 기본 주소 체계 규칙 때문입니다. 깃허브에서 무료로 제공하는 웹사이트의 기본 주소는 https://[본인아이디].github.io 형식으로 고정됩니다. 즉, 저장소(Repository) 이름을 아이디와 똑같이 만들었다면 뒤에 /profile을 붙이지 않아야 정상적으로 접속됩니다. 현재 상황에서 사이트가 열리지 않는 원인과 해결 방법을 두 가지로 나누어 점검해 드리겠습니다. 🔍 원인 분석 및 해결 방법 1. 주소에서 /profile을 제외하고 접속해 보세요 (가장 유력) 강사님의 깃허브 아이디가 000yoryeong이고, 저장소 이름을 규칙대로 000yoryeong.github.io로 만드셨다면 실제 인터넷 주소는 다음과 같습니다. 올바른 주소: https://000yoryeong.github.io 뒤에 붙은 /profile을 지운 위 주소로 다시 한번 브라우저창에 입력해 보세요. 2. 저장소(Repository) 이름 자체를 다시 확인해 보세요 만약 깃허브에서 새 저장소를 만들 때 이름을 profile이라고 지정하셨다면 주소 체계가 달라집니다. 저장소 이름이 profile인 경우의 주소: https://000yoryeong.github.io/profile이 맞습니다. 이 경우 활성화 방법: 깃허브 내 profile 저장소로 들어가신 후, [Settings] ➡️ 좌측 메뉴의 [Pages] ➡️ Build and deployment 항목의 Branch가 None에서 main(또는 master)으로 선택되어 있고 우측의 [Save]를 눌렀는지 확인해야 합니다. 이 설정이 꺼져 있으면 주소가 맞아도 사이트가 나오지 않습니다. 💡 가장 추천하는 깔끔한 세팅법 강사님의 공식 프로필 사이트인 만큼, 주소 창에 아이디만 입력했을 때 바로 대문 페이지가 뜨는 1번 방식이 가장 전문성 있어 보입니다. 깃허브에 접속하여 현재 만든 저장소 이름을 000yoryeong.github.io로 수정(또는 새로 생성)합니다. 그 저장소 안에 파일명을 반드시 소문자로 index.html이라 적고 아까 전해드린 코드를 넣습니다. 잠시 후 https://000yoryeong.github.io로 접속하시면 아임웹 스타일의 원페이지 프로필이 깔끔하게 나타납니다. 현재 깃허브 저장소(Repository) 이름을 어떤 것으로 만드셨는지 말씀해 주시면, 해당 이름에 맞는 정확한 주소와 설정법을 바로 잡아드리겠습니다. 내가 이미 다른 앱들을 만들어서 보내주신 대시보드 스크린샷을 보니 이미 cat-health-app, cbt 같은 개별 프로젝트 저장소들을 훌륭하게 관리하고 계셨군요. 어떤 상황인지 명확히 이해했습니다. 현재 대시보드에 000yoryeong/profile이라는 이름으로 저장소(Repository)가 만들어져 있기 때문에, 깃허브 시스템은 이 사이트의 주소를 https://000yoryeong.github.io/profile로 인식하게 됩니다. 다만, 이 주소로 접속했을 때 화면이 나오지 않는 이유는 해당 저장소 안에서 'GitHub Pages(웹 배포)' 기능이 활성화되지 않았기 때문입니다. 이미 만들어두신 profile 저장소를 그대로 활용하여 3분 안에 사이트를 정상적으로 띄우는 명확한 체크포인트를 안내해 드릴게요. 딱 두 가지만 세팅하시면 됩니다. 🛠️ 000yoryeong/profile 활성화 체크포인트 1단계: 파일 이름 확인 (index.html) 대시보드에서 000yoryeong/profile 저장소를 클릭해 안으로 들어갑니다. 저장소 내부에 들어가 있는 메인 소스코드 파일 이름이 대문자가 섞인 Index.html이거나 다른 이름이 아닌, 반드시 전부 소문자인 index.html로 되어 있는지 확인해 주세요. 서버 시스템은 대소문자를 엄격하게 구분하므로, 반드시 소문자여야 대문 페이지로 자동 인식합니다. 2단계: 깃허브 페이지 배포 스위치 켜기 (핵심) 현재 이 과정이 누락되어 사이트가 열리지 않는 것입니다. 아래 순서대로 메뉴를 클릭해 주세요. profile 저장소 화면 상단의 톱니바퀴 아이콘 [Settings]를 누릅니다. 좌측 사이드바 메뉴에서 [Pages] 메뉴를 찾아서 클릭합니다. 중간에 있는 Build and deployment 섹션을 확인합니다. Branch 항목이 현재 None으로 되어 있을 것입니다. 이 부분을 클릭하여 main (또는 master)으로 변경합니다. 바로 옆의 폴더 선택 칸은 / (root) 그대로 둔 채, 우측의 [Save] 버튼을 누릅니다. 🌐 세팅 완료 후 확인하기 [Save] 버튼을 누르고 나면 화면 상단에 파란색이나 녹색으로 배포 진행 상황이 표시됩니다. 약 1~2분 정도 깃허브 서버가 빌드할 시간을 준 뒤, 아래 주소로 접속해 보세요. 👉 https://000yoryeong.github.io/profile 이제 아임웹 스타일로 짜인 단정하고 세련된 원페이지 프로필이 정상적으로 나타날 것입니다. 기존에 생성해 두신 다른 앱 저장소들(cat-health-app, cbt)에는 전혀 영향을 주지 않으니 안심하고 세팅하셔도 됩니다. 혹시 설정을 바꿨는데도 다른 에러 메시지가 뜬다면 어떤 화면이 나오는지 말씀해 주세요. https://000yoryeong.github.io/profile/ 여기에서 처음에 최종학력 부분은 삭제하고, 맨 마지막에 첨부파일을 추가하고 싶고, (첨부파일 링크도 좋음), 이메일보내기와 블로그 바로가기 버튼이 있으면 좋겠어 요청하신 대로 1페이지 상단 히어로 섹션에서 최종학력을 삭제하여 군더더기 없이 실무 역량에 집중되도록 고쳤습니다. 또한, 하단에 이력서 다운로드(첨부파일) 섹션을 새롭게 추가하고, 상단과 하단에 각각 [이메일 보내기]와 [블로그 바로가기] 버튼을 세련된 아임웹 감성의 UI 컴포넌트로 디자인하여 배치했습니다. 아래의 수정된 전체 코드를 복사하셔서, 기존 깃허브 profile 저장소의 index.html 파일 내용을 전체 선택(Ctrl+A) 후 덮어쓰기(Ctrl+V) 하시면 즉시 반영됩니다. 🛠️ 기능이 추가된 index.html 전체 소스코드 HTML 윤정연 | 콘텐츠 기획 PD
